# Taller Kotlin — Mini Retro

## ¿Qué fue lo más difícil?
Lo más difícil fue entender y aplicar correctamente la nullabilidad y el uso de lambdas, porque obligan a pensar en los posibles valores null y en cómo pasar funciones como parámetros sin generar errores.

---

## ¿Qué concepto aplicaría en una app Android real?
Aplicaría la nullabilidad y las clases (POO) para modelar datos de usuarios y productos de forma segura, evitando crashes cuando un dato no esté disponible y organizando mejor la lógica de la aplicación.

---

## 3 aprendizajes clave

- Uso correcto de `val` y `var` para controlar la mutabilidad.
- Manejo seguro de null con `?`, `?.` y `?:`.
- Uso de funciones y lambdas para reutilizar y parametrizar comportamiento.
